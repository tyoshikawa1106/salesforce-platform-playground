# 顧客満足度アンケート

## 概要

サービスへの満足度を5段階で収集し、任意のコメントを受け付ける1ページ構成のSalesforce Surveysアンケートフローです。

## 目的・利用場面

Salesforce Surveysから案内した回答者に、サービス評価と自由記述のフィードバックを入力してもらう場面で利用します。

## 対象実装・メタデータ

| 種別 | API 名                  | 実装上の表示名          | 役割                           |
| ---- | ----------------------- | ----------------------- | ------------------------------ |
| Flow | `customer_satisfaction` | `Customer Satisfaction` | アンケート画面と回答項目の定義 |

## 入力

- サービス評価: 「悪い」を表す1から「良い」を表す5までの数値選択。任意入力です。
- コメント: 文字列の自由入力。任意入力です。
- Survey実行コンテキスト: `invitationId`、`guestUserLang`、`previewMode`を入出力変数として受け取ります。
- 評価項目の初期値を文字列型の入力変数で受け取り、数値型の評価コンポーネントへ渡します。

## 処理内容

1. 回答者に評価とコメントの入力画面を表示します。
2. 回答者が完了すると、Surveyランタイムに回答を引き渡します。
3. 完了画面に参加へのお礼を表示します。

実装されている英語表示と日本語での意味は次のとおりです。

| 項目             | 実装上の表示                       | 日本語での意味                   |
| ---------------- | ---------------------------------- | -------------------------------- |
| 評価の質問       | `How would you rate our service?`  | サービスをどのように評価しますか |
| 最低評価の選択肢 | `Bad`                              | 悪い                             |
| 最高評価の選択肢 | `Good`                             | 良い                             |
| コメントの質問   | `Any comments or feedback for us?` | コメントやご意見はありますか     |
| 完了メッセージ   | `Thanks for participating!`        | ご回答ありがとうございました     |

## 出力・更新対象

- Flow変数としてSurvey実行コンテキストと完了メッセージを出力します。
- 回答の永続化はSalesforce Surveysのランタイムが担当します。このFlow内に明示的なレコード作成、更新、削除要素はありません。

## 権限・実行条件

- Salesforce Surveysを利用でき、Surveyの回答ページを実行できることが前提です。
- Flow の状態は有効（`Active`）です。

## エラー処理

Flow内に独自のFault経路はありません。表示や回答保存のエラーはSalesforce Surveysの標準動作に従います。

## 関連コンポーネント

- Salesforce Surveysの招待、回答、プレビュー実行
- Survey標準画面コンポーネント `survey:runtimeRating`

## テスト・確認観点

- このFlowを直接検証する自動テストはリポジトリにありません。次の観点をSalesforce Surveys上で確認します。
- 1から5までの評価を選択して完了できること
- 評価またはコメントを未入力でも完了できること
- コメントを入力した回答がSurvey側に保存されること
- 招待経由、ゲスト回答、プレビューで画面を表示できること
- 完了時にサンクスメッセージが表示されること

## 制約・注意事項

- 実際のアンケート画面では、質問文、選択肢、完了メッセージが英語で表示されます。
- 評価とコメントはいずれも必須ではありません。
- 回答保存の詳細はSurvey標準機能に依存し、Flowメタデータだけでは完結しません。

## 既知の差異・確認事項

- 状態: 現行メタデータ確認済み、承認済み要求との差異は未判定
- 現行実装は `customer_satisfaction.flow-meta.xml` から確認しています。
- 承認済み要求またはSalesforce Surveysの運用要件の管理元をリポジトリ内で確認できないため、要求との差異は判定していません。
