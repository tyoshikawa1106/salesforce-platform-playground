# 取引先名の法人格正規化

## 概要

取引先の作成時と名称変更時に、取引先名に含まれる株式会社、有限会社の略称を正式名称へ置換します。

## 入力

- 作成時: `Trigger.new` の取引先
- 更新時: `Trigger.new` と `Trigger.oldMap` の取引先
- 対象項目: `Account.Name`
- 置換対象:
    - `㈱`、`（株）`、`(株)`
    - `㈲`、`（有）`、`(有)`

## 処理内容

1. 作成時は、Nameが空でないすべての取引先を対象にします。
2. 更新時は、Nameが変更された取引先だけを対象にします。
   `oldMap` または対応する更新前レコードを取得できない場合は、変更済みとして正規化します。
3. 株式会社の3種類の略称を、元の位置を保ったまま `株式会社` へ置換します。
4. 有限会社の3種類の略称を、元の位置を保ったまま `有限会社` へ置換します。
5. beforeコンテキストのレコードへ直接反映します。

## 出力・更新対象

- `Account.Name` の正規化後の値
- Trigger内で追加のDMLは実行しません。
- 略称を含まない文字列と、更新時に変更されていないNameは変更しません。

## 権限・実行条件

- `AccountTrigger` の `before insert` または `before update` から実行します。
- 更新時は `Account.Name` が変更されたレコードだけを正規化します。
- 複数レコードを一括処理でき、追加のSOQLやDMLは実行しません。

## エラー処理

独自の例外処理はありません。予期しない例外が発生した場合は取引先の保存処理全体が失敗します。

## テスト・確認観点

- `AccountTriggerHandlerTest` で、3種類の株式会社略称と3種類の有限会社略称が正式名称へ置換されること
- 前方、途中、後方にある略称の位置を保って置換されること
- 1つの名称に複数の略称が含まれる場合にすべて置換されること
- 略称を含まない名称が変更されないこと
- `AccountTriggerServiceTest` で、Name未変更、null、空白、空リスト、更新前レコード不在を安全に処理すること

## 制約・注意事項

- 対象は株式会社と有限会社の定義済み6表記だけです。
- 合同会社、合資会社、一般社団法人などは正規化しません。
- 略称の位置を法人格として妥当か判定せず、文字列中に存在すれば置換します。
- 正規化済みの `株式会社`、`有限会社` は変更しません。

### 大量データとGovernor Limits

- 入力リストを1回走査する文字列置換であり、追加のSOQLやDMLは実行しません。
- Trigger経由の251件一括登録を `AccountTriggerHandlerTest` で検証します。

## 既知の差異・確認事項

- 状態: 現行実装確認済み、承認済み要求との差異は未判定
- 現行実装は `AccountTriggerService` と `AccountTriggerHandlerTest`、`AccountTriggerServiceTest` から確認しています。
- 承認済み要求または外部契約の管理元をリポジトリ内で確認できないため、要求との差異は判定していません。
